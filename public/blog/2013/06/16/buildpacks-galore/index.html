Jun 16th, 2013

We recently started tracking the BUILDPACK_URL environment variable in Heroku deploys.

Observations

  • Some buildpacks are tarballs (.tgz or tar.gz) on S3. This is presumably faster than hitting a github repo.
  • The practice of appending a git revision #fragment to the buildpack URL seems widely adopted. Branches FTW!

Arbitrary metadata for deployed apps is stored in a Postgres database using hstore. Here’s the query I used to fetch the list:

select attributes->'buildpack_url' as buildpack, count(*) as deploys
from all_events
where attributes ? 'buildpack_url'
group by buildpack
order by deploys desc;
